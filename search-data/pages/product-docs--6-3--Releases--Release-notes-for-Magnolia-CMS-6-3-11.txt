# Release notes for Magnolia CMS 6.3.11

[LTS release](#support:ROOT:index.adoc) • Delivered on July 18, 2025 • Page updated on July 24, 2025 • Changelog: [18427](https://magnolia-cms.atlassian.net/issues/?filter=18427)

> **Warning:** Do not upgrade to Magnolia 6.3.11.This version contains two issues that can impact the stability and functionality of your Magnolia instances:Upgrading to Magnolia 6.3.9, 6.3.10, 6.3.11, or 6.3.12 initiates a Marketing Tags version handler task that may cause instances to become unresponsive if the repository contains a large number of versions. See the Known issues page for more information.The issue has been resolved in the Magnolia 6.3.13 release.An unexpected change introduced by ticket MGNLREST-877 breaks the Nodes API functionality. This issue also affects Put/Post operations, resulting in errors such as:Unrecognized field 'properties' (class info.magnolia.rest.service.node.v1.RepositoryNode), not marked as ignorableWorkaround: No workaround is currently available.The issue is resolved in Magnolia 6.3.12, where MGNLREST-877 is reverted.

**Magnolia 6.3.11** release introduces key improvements and bug fixes to enhance functionality, stability, and security. Notable updates include optimized handling of response filters for personalization endpoints and improved thumbnail and preview support for non-image assets.

Bug fixes address issues such as Vaadin Atmosphere log handling, redirect template functionality with special characters, and unintended page deletion during component removal.

## [](#_bug_fixes)Bug fixes

-   [ADMINCTR-586](https://magnolia-cms.atlassian.net/browse/ADMINCTR-586) Vaadin Atmosphere logs not handled by SLF4J

-   [MAGNOLIA-9662](https://magnolia-cms.atlassian.net/browse/MAGNOLIA-9662) Redirect template not working when target page name contains umlaut (¨) characters

-   [MGNLREST-876](https://magnolia-cms.atlassian.net/browse/MGNLREST-876) Decorations not working for the Cache module

    Decorating configuration files must have `config` in the filename in Magnolia 6.3, for example, `/rest-services/decorations/cache/config.yaml` or `/rest-services/decorations/cache/config.contentCaching.yaml`. This was optional in Magnolia 6.2.

-   [MGNLUI-9210](https://magnolia-cms.atlassian.net/browse/MGNLUI-9210) A page may be unintentionally deleted during a **Delete component** action on a slow connection
