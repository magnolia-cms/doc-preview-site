# List field

`ListSelectFieldDefinition` renders a list box where more than one item can be selected. If the number of items exceeds the height of the component, a scrollbar will appear.

> **Note:** The comboBoxField and listSelectField field types don’t support natural ordering. Instead, use the linkField with textInputallowed set to true as the better choice for ordering items alphabetically.For JSON data sources, use the JsonComboBoxFieldDefinition.

`ListSelectFieldDefinition` extends the `AbstractMultiselectFieldDefinition` class.

## [](#_example_definition)Example definition

```yml
listSelect:
  $type: listSelectField
  label: Books
  datasource:
    $type: jcrDatasource
    workspace: books
    describeByProperty: name
    allowedNodeTypes:
      - lib:book
```

> **Tip:** You can preview this example using the field-examples-module. See Field examples module for more information.

## [](#_field_properties)Field properties

### [](#_field_specific_properties)Field-specific properties

| Property | Description |
| --- | --- |
| `rows` | **optional**, *default is `4`*<br><br>Number of rows in the field. When set to `0`, the actual number of rows is determined implicitly by the field. |

### [](#_common_select_field_properties)Common select field properties

| Property | Description |
| --- | --- |
| `datasource` | **required**<br><br>Connects the field to a [data source](../../../../../../Apps/App-configuration/Data-source-definition/). Options are populated via the configured data source.<br><br>Use the fully qualified class name or the `$type` alias if the definition class is annotated with `info.magnolia.ui.datasource.DatasourceType`. |
