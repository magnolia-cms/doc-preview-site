# Ingresses help

An ingress is a Kubernetes resource that manages external access to services, by routing traffic to the appropriate pods based on defined rules. Use ingresses to expose services securely and efficiently.

Adding an ingress via the Cockpit simplifies the process, eliminating the need for manual Rancher imports or complex `.yaml` configurations. Create and claim an ingress in the Cockpit. This ensures a managed contract where updates are tracked and external changes are ignored, enhancing security, consistency, and ease of maintenance
