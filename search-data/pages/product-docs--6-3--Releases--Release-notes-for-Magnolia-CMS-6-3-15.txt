# Release notes for Magnolia CMS 6.3.15

[LTS release](../../../../support/) • Delivered on September 16, 2025 • Page updated on October 15, 2025 • Changelog: [19582](https://magnolia-cms.atlassian.net/issues/?filter=19582)

> **Warning:** Avoid upgrading to Magnolia 6.3.15 or 6.3.16 in production environments.Instead, upgrade to Magnolia 6.3.17 or a later version.Issue summary: A UI freeze in the Rich Text editor (CKEditor 5) occurs, especially when switching tabs or languages, disrupting routine content editing.The fix is tracked in the ticket MGNLUI-9284.

**Magnolia 6.3.15** is a maintenance release focused on stability and usability for developers and administrators, primarily enhancing CKEditor 5 integration and fixing content translation issues. Several security improvements are also included in this release.

## [](#_improvements)Improvements

### [](#_enhanced_styles_in_ckeditor_5)Enhanced styles in CKEditor 5

Vaadin-compatible styling is moved to the UI framework module. This ensures that only the Vaadin `richTextField` is affected by the Vaadin-compatible styling. It won’t impact the `richTextField` in [accessible forms](../../Developing/Templating/Dialog-definition/Accessible-forms/) or other presentation layers where `richTextField` instances also rely on CKEditor 5.

[MGNLUI-9242](https://magnolia-cms.atlassian.net/browse/MGNLUI-9242)

### [](#_security_improvements)Security improvements

[BUILD-1311](https://magnolia-cms.atlassian.net/browse/BUILD-1311), [BUILD-1446](https://magnolia-cms.atlassian.net/browse/BUILD-1446), [BUILD-1467](https://magnolia-cms.atlassian.net/browse/BUILD-1467), [BUILD-1470](https://magnolia-cms.atlassian.net/browse/BUILD-1470), [MAGNOLIA-9688](https://magnolia-cms.atlassian.net/browse/MAGNOLIA-9688), [MGNLCKE-49](https://magnolia-cms.atlassian.net/browse/MGNLCKE-49), [MGNLFORM-409](https://magnolia-cms.atlassian.net/browse/MGNLFORM-409), [MGNLREST-880](https://magnolia-cms.atlassian.net/browse/MGNLREST-880)

We keep the details of security fixes private in line with our [security policy](../../Administration/Security/Security-policy/). Stay informed on the latest security updates by subscribing to our Security Announcements list. Contact our [Support team](../../../../support/) if you need more information.

#### [](#_auditloggingmanager_implementation_cannot_be_overridden)`AuditLoggingManager` implementation cannot be overridden

[MAGNOLIA-9688](https://magnolia-cms.atlassian.net/browse/MAGNOLIA-9688) introduces a warning to indicate that the `AuditLoggingManager` implementation cannot be overridden by other modules. To interact with audit logging events, use event listeners instead of attempting to modify the implementation directly.
