# Admin Groups deep dive

## [](#_cockpit)Cockpit

View the predefined groups that control access to Cockpit features. These groups are managed by Magnolia and cannot be edited. Users assigned to these groups gain specific permissions within the Cockpit interface.

## [](#_rancher)Rancher

View the predefined groups that control access to Rancher, the underlying Kubernetes cluster management platform. These groups are managed by Magnolia and cannot be edited. Users assigned to these groups can perform infrastructure-level operations.

## [](#_magnolia_admincentral)Magnolia (Admincentral)

Manage groups that control access to Magnolia Admincentral. Create, edit, and delete groups to organize users and assign roles that determine what actions they can perform within the Magnolia Admincentral.

## [](#_custom)Custom

Create your own groups to organize users according to your needs. Custom groups can be assigned both PaaS Client API roles for Cockpit API access and Rancher roles for cluster management. Use custom groups to define role combinations that match your organizationâ€™s access requirements.
