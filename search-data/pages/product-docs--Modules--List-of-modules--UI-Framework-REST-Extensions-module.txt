# UI Framework REST Extensions module

| --- | --- |

| Latest | 3.0.0 |

The Magnolia UI Framework REST Extensions module is a DX Core Edition feature which complements the [REST Client module](../REST-Client-module/) by enabling configured REST clients as:

-   Datasources of subapps and UI elements such as the [link field](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Link-field/) or [Combobox field](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Combobox-field/).

-   [Search result suppliers](REST-search-result-suppliers-for-Global-Search/) for the [Global Search](../../../Authoring/Search/Global-search/).


## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../Developing/Bundles-and-webapps/):

```xml
<dependency>
  <groupId>info.magnolia.restclient</groupId>
  <artifactId>magnolia-rest-client-ui</artifactId>
  <version>3.0.0</version> (1)
</dependency>
```

|     |     |
| --- | --- |
| **1** | Should you need to specify the module version, do it using `<version>`. |

### [](#_handling_offline_datasources)Handling offline datasources

The module provides two additional [field types](../../../Developing/Templating/Dialog-definition/Field-definition/#_field_types) to handle offline datasources:

-   `jsonLinkField`

-   `jsonComboBoxField`


These types wrap the `linkField` and `comboBoxField` types, respectively, and check whether REST calls are working properly. If an error occurs, a label appears next to the component that uses the REST field type and the component is disabled.
