# Pages module

[Content management](../../../use-cases/content-management/) Bundled: Community edition

| Edition | CE  |
| --- | --- |

| Latest | 6.3.7 |
| [Available on SPA/Headless](../../../Developing/Templating/Freemarker-vs-headless/#_magnolia_modules_and_features) | Yes |

The Pages module installs the [Pages app](../../../Apps/List-of-apps/Pages-app/) that allows editors to editors to edit Web pages. See [Authoring](../../../Authoring/) for more information about using the app.

For [Single-page applications](../../../Features/Single-page-applications/), the `magnolia-spa-rendering` submodule provides a dedicated [SPA template definitions endpoint](../../../Developing/API/SPA-templating-API/#_spa_template_definitions_endpoint), which is used to deliver page, area and component template definitions.

> **Tip:** See an overview of differences between FreeMarker and headless templating.

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../Developing/Bundles-and-webapps/):

```xml
<dependency>
  <groupId>info.magnolia.pages</groupId>
  <artifactId>magnolia-pages-app</artifactId>
  <version>6.3.7</version> (1)
</dependency>
```
