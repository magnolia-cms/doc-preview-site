## [](#_using)Using

You can use Magnolia CLI and its plugins to:

-   Download a project with a Magnolia webapp.

-   Quickly create a [light module](../product-docs/6.3/Modules/Light-modules/) skeleton, with the folders and files that form a Magnolia light module.

-   Generate [light module items](../product-docs/6.3/Modules/Light-modules/#_what_items_can_be_defined_in_a_light_module).

-   Start a Magnolia webapp.


## [](#_versions)Versions

### [](#_supported)Supported

[**Magnolia CLI v5**](getting-started/), a plugin-based CLI designed for developers seeking modularity and the ability to extend functionalities through custom CLI commands.

### [](#_end_of_life)End-of-life

[**Magnolia CLI v4**](#4.x@magnolia-cli:ROOT:magnolia-cli.adoc), a command-based CLI, reached end-of-life (EOL) on April 16, 2025. Itâ€™s no longer maintained and will not receive any updates, including security fixes. For continued support, improvements, and compatibility with current Magnolia versions, use CLI version 5.

Version 4 includes the legacy *Magnolia 5 UI* prototypes, which are marked with the `-m5` suffix.

> **Note:** CLI versions 2 and 3, while still available for installation through NPM, are no longer supported.
