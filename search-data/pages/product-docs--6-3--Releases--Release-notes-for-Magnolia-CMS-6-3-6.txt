# Release notes for Magnolia CMS 6.3.6

[LTS release](../../../../support/) • Delivered on March 24, 2025 • Changelog: [16217](https://magnolia-cms.atlassian.net/issues/?filter=16217)

**Magnolia 6.3.6** is primarily a bug-fixing and security release that delivers critical security fixes. This release also enhances workflows with .zip file extraction, preserving folder structures for efficient asset management, and adds REST client caching for improved performance.

We keep the details of the security fixes private in line with our [security policy](../../Administration/Security/Security-policy/). Contact our [Support team](../../../../support/) if you need more information.

## [](#_improvements)Improvements

### [](#_zip_extraction_of_assets_upon_upload)ZIP extraction of assets upon upload

You can now upload and extract .zip files while preserving the original file and folder structures. This enhancement simplifies uploading and organizing assets efficiently, especially when dealing with big data sets.

-   Recreate nested folder structures in the `dam` workspace.

-   Streamline uploading structured content.

-   Improve efficiency for teams managing large asset collections.


[MGNLDAM-1586](https://magnolia-cms.atlassian.net/browse/MGNLDAM-1586) Ability to upload nested folder structures and extract (ZIP)

### [](#_caching_in_the_rest_client)Caching in the REST client

Similar to Magnolia 6.2, the 6.3 branch now supports REST client definition caching configuration.

-   [MGNLRESTCL-232](https://magnolia-cms.atlassian.net/browse/MGNLRESTCL-232) Re-introduce cache configuration in REST client definition
