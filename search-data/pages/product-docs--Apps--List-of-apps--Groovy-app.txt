# Groovy app

The Groovy app is installed by the Groovy module that adds Groovy capabilities to Magnolia. For more information about using the app see the [Groovy module](../../../Modules/List-of-modules/Groovy-module/) documentation.

![Groovy scripts edit with the action bar](../../../_images/groovy-scripts.png) ![Groovy console](../../../_images/groovy-console.png)

## [](#_configuration)Configuration

The Groovy app is installed by the [Groovy module](../../../Modules/List-of-modules/Groovy-module/). The configuration of the `browser` and `detail` subapps is typical of any content app. The `console` subapp uses the custom subapp class info.magnolia.module.groovy.console.GroovyConsoleSubApp. Permission to access the app is limited to users assigned the `scripter` [role](../../../Administration/IAM/Roles-and-access-control-lists/). In addition, the Dev app group that this app belongs to is only available to the `superuser` role. This is configured in the [app launcher layout](../../../Administration/App-Launcher/App-launcher-layout/). The app is configured in `/modules/groovy/apps/groovy`.

## [](#_workspace)Workspace

The Groovy app operates on the `scripts` workspace that stores the Groovy scripts.

## [](#_node_types)Node types

The Groovy app operates on `mgnl:folder` and `mgnl:content` [node types](../../../Administration/Architecture/Node-types/).

## [](#_groovy_shell_scripts)Groovy shell scripts

There are several examples of groovy scripts available in our [git repository](https://bitbucket.org/magnolia-cms/groovy-shell-scripts).
