# CTSX DeepL Translator

[Content management](../../product-docs/RNs-6-4-3/use-cases/content-management/) [Unbundled: Extension](https://gitlab.magnolia-platform.com/pd/extensions/ps-maintained/content-translation-support-ext) Version 5.1.0 [Magnolia 6.4](../../product-docs/RNs-6-4-3/) compatible

The **CTSX DeepL** module performs translations using the DeepL APIs. Translations are handled synchronously.

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../product-docs/RNs-6-4-3/Developing/Bundles-and-webapps/):

> **Important:** In order to start the DeepL translation process, please make sure that the core and apps modules in the Content Translation Extended have already been installed.

```xml
<dependency>
  <groupId>info.magnolia.translation</groupId>
  <artifactId>magnolia-content-translation-support-ext-deepl</artifactId>
  <version>5.1.0</version>
</dependency>
```

| Property | Description |
| --- | --- |
| `class` | **required** Definition class for the translator to be used. See `info.magnolia.translation.ext.provider.deepl.definition.DeepLProviderDefinition` . |
| `clientId` | **required** Unused but needs a value. |
| `clientToken` | **required** token from DeepL account |
| `glossaryId` | **optional** Specify the glossary to use for the translation. |
| `configName` | **required** Identifies the configuration within Magnolia. |
| `defaultFlag` | **optional** The default configuration used by the system in the case of multiple translation services being used. |
| `enabledFlag` | **required** Enable or disable the configuration using this flag. |
| `implementationClass` | **required** Implementation class for interfacing with DeepL. See `info.magnolia.translation.ext.provider.deepl.translator.DeepLTranslator` . |
| `extraQueryParams` | **optional** You can adjust more parameters for document translation with this feature. |

## [](#_usage)Usage

After registration and starting, The Content Translation Support Extended (CTSX) module will bind the DeepL translator to Configuration field.

> **Note:** You can test the connection to the translator api before using.

### [](#_how_it_works)How it works

Use the same steps outlined with [CTSX Google Translator](../google/) but you will choose **DeepL** instead of Google.com in the Configuration.
