# Installing Magnolia through npm CLI

> **Caution:** If you set a new superuser password in a properties file before starting up a fresh installation, you must delete the password from the properties file. This is necessary to avoid a security vulnerability. Ensure any bootstrapped password is removed after logging in the first time.magnolia.properties...
> magnolia.superuser.password=<YOUR_SUPERUSER_PASSWORD> (1)
>
> magnolia.eric.password=<BOOTSTRAPPED_PASSWORD_FOR_ERIC> (2)
> magnolia.eric_de.password=<BOOTSTRAPPED_PASSWORD_FOR_ERIC_DE> (2)
> magnolia.peter.password=<BOOTSTRAPPED_PASSWORD_FOR_PETER> (2)
> magnolia.tina.password=<BOOTSTRAPPED_PASSWORD_FOR_TINA> (2)1See values.yml for an example configuration on DX Cloud.2These users are exclusive to Magnolia demo bundles. Their passwords are not part of the bundle. For initial access, you may need to bootstrap passwords. After the first login, ensure that you remove these passwords from the properties file.

## [](#_logging_into_the_author_instance)Logging into the Author instance

Go to [`http://localhost:8080/magnoliaAuthor`](http://localhost:8080/magnoliaAuthor) and log in to the Author instance as a `superuser` :

-   Username: `superuser`

-   Password: `superuser`


The first page you see is the `App Launcher`, in which you can see groups of apps:

![App launcher with grouped apps](../../../_images/mgnl-after-login.png)

Congratulations. Magnolia is now installed and running.
