# Date field

`DateFieldDefinition` renders a calendar that allows the user to select a date and optionally a time.

> **Note:** This date field definition is part of the Magnolia 6 UI framework. The fully qualified class name is info.magnolia.ui.field.DateFieldDefinition.

## [](#_example_definition)Example definition

```yml
date:
  $type: dateField
  label: Date
  type: java.util.Date
  time: true
```

> **Tip:** You can preview this example using the field-examples-module. See Field examples module for more information.

| Property | Description |
| --- | --- |
| `dateFormat` | **optional** , *default is yyyy-MM-dd* Format in which the date is displayed to users. See the [DateTimeFormatter](https://docs.oracle.com/javase/10/docs/api/java/time/format/DateTimeFormatter.html) class for allowed patterns. If `time` is set to `true` ( [reference](#time) ), the time is added to the date displayed to users. In this case, the default is `yyyy-MM-dd HH:mm` . Examples `dateFormat: yyyy-MM-dd (1) dateFormat: yyyy-MM-dd HH:mm (2) dateFormat: yyyy-MMMM-dd # 2026-October-31 dateFormat: EEE, MMM d, yyyy # Tue, Oct 31, 2026` ** **1** Default value. **Example** : `2026-10-31` ** **2** Default value when `time` is set to `true` . **Example** : `2026-10-31 15:09` | ** **1** | Default value. **Example** : `2026-10-31` | ** **2** | Default value when `time` is set to `true` . **Example** : `2026-10-31 15:09` |
| ** **1** | Default value. **Example** : `2026-10-31` |
| ** **2** | Default value when `time` is set to `true` . **Example** : `2026-10-31 15:09` |
| `defaultValue` | **optional** , *default is now* Set a specific date and time such as `"2026-10-31"` or use `now` for the current date and time on the host server. Magnolia stores dates and times using server time. The stored dates and times are converted and displayed to users in their [preferred time zone](../../../../../../Administration/IAM/Users/) . The default value applies only to new content items, not to those that have already been created. Enclose the property value in quotes when using YAML: `defaultValue: "2026-10-31" defaultValue: "2026-10-31 15:09"` If you define the `dateFormat` and `timeFormat` properties, provide `defaultValue` using the same formats: `dateFormat: MM/dd/yyyy timeFormat: h:mm a defaultValue: "10/31/2026 3:09 PM"` |
| []() `time` | **optional** , *default is false* Enables time selection. Renders dropdown lists at the bottom of the calendar. |
| `timeFormat` | **optional** , *default is HH:mm* Format in which the time is displayed to users. See the [DateTimeFormatter](https://docs.oracle.com/javase/10/docs/api/java/time/format/DateTimeFormatter.html) class for allowed patterns. Examples `timeFormat: HH:mm # 15:09 timeFormat: h:mm a # 3:09 PM` |
