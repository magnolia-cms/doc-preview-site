# Magnolia Database deep dive

## [](#_sidebar_filters)Sidebar Filters

The sidebar filters allow you to refine your view by selecting specific criteria.

-   **Date range**: Select the time period using preset ranges or the calendar picker for custom dates.

-   **Memory unit**: Choose how to display storage sizes (KB, MB, GB, or TB).

-   **Release**: Select the namespace to analyze. Defaults to the first available option.

-   **Pod**: Select the specific pod instance to analyze. Defaults to the first available option.


Click **Apply filters** to update the view.

## [](#_database)Database

View overall database health and performance metrics.

-   **Total size**: Current database size and growth trends over time.

-   **Binary size**: Storage used by assets, documents, and other large records.

-   **Average binary size**: Typical size of individual binary records.

-   **Cache ratio**: Database cache efficiency, where higher values indicate better performance.


## [](#_workspaces)Workspaces

Examine storage metrics for each JCR workspace.

The table shows records, total size, table size, index size, and dead tuples for each workspace, sorted alphabetically.

Click any metric value to view a detailed time series chart for that workspace.
