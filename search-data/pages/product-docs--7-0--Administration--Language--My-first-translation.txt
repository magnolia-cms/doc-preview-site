# My first translation

In this tutorial, you translate the user interface (UI) of a Magnolia app to another language by creating a new i18n properties file for the language and adding the file to the app. Don’t worry if you’re not a developer. You don’t need a development environment – all you need is language skills and Magnolia.

The app whose UI labels and messages you are going to translate is the Bookshelf app, created in the [My first content app](../../../Apps/Developing-an-app/My-first-content-app/) tutorial. The labels and messages are in English and in this tutorial you [internationalize](../Internationalization/) the app for Spanish-speaking users.

## [](#_prerequisites)Prerequisites

The Bookshelf app is meant to be run as an app in Magnolia, so first you must install Magnolia. Before installing Magnolia, check that the following system components are already installed on your system.

> **Note:** The CLI installation steps are based on Magnolia CLI v4.

## [](#_get_node_js)Get node.js

Magnolia CLI is an [npm](https://www.npmjs.com/) package providing a command line interface (CLI) tool to set up and facilitate light development with Magnolia. The Magnolia CLI tool runs on [Node.js](https://nodejs.org/). If you do not have Node.js installed, go to [Node.js](https://nodejs.org/) and download and install the latest LTS version.

To check the version of your node installation, run the following command in a shell:

node -v

### [](#_git)Git

In this tutorial, you need [Git](https://git-scm.com/) to clone the Bookshelf app from our repository into your local Magnolia installation. You can check that Git is installed on your system by executing the `git --version` command. ## [](#_installing_and_starting_magnolia)Installing and starting Magnolia

Now install Magnolia.

## [](#_installing_magnolia_with_cli_only)Installing Magnolia with CLI only

Use the `jumpstart` command to install Magnolia. This command downloads, unpacks and pre-configures a Magnolia bundle of your choice.

> **Tip:** To install a specific Magnolia version, use the -m flag, for example:npx @magnolia/cli@latest jumpstart --magnolia "6.3.0"To see a list of options available for any Magnolia CLI command, use the command together with the -h flag, for example:npm run mgnl -- create-page -h

Windows users using [PowerShell](https://learn.microsoft.com/en-us/powershell/) must enclose the two consecutive hyphens (`--`) in quotes:

```console
npm run mgnl "--" ...
```

To install Magnolia:

1.  Change to the directory to where you want to install the Magnolia bundle. For example:

    ```console
    cd /Users/<username>/dev/
    ```

2.  Execute the Magnolia CLI [`jumpstart`](../../../../../magnolia-cli/plugins/available-plugins/jumpstart-plugin/) plugin command:

    ```console
    mgnl jumpstart
    ```

3.  Choose the `demo-webapps` template, followed by the `magnolia-community-demo-webapp` template containing Magnolia Community Edition bundled with the [Travel Demo](../../../../6.3/Getting-started-with-Magnolia/Demo/) and a Tomcat server. It creates folders for the Tomcat server and for the light modules according to the CLI configuration.

    Once the setup operation is complete, you should see a message similar to this one:

    ```console
    info: Project successfully downloaded
    info: Run "npm run mgnl" to see the list of registered commands
    info: Start the project using: "npm run mgnl -- start"
    info: If the "StartPlugin" is not available, install it using: "npm run mgnl -- add-plugin @magnolia/cli-start-plugin"
    info: For additional plugins and features, refer to the Magnolia CLI documentation: https://docs.magnolia-cms.com/magnolia-cli
    ```


## [](#_starting_magnolia_with_cli_mgnl_start)Starting Magnolia with CLI mgnl start

Use the `mgnl start` command to start Magnolia. In the parent directory of `light-modules`, enter:

mgnl start

The command installs and starts Magnolia. This is complete when you see a message like `Server startup in 112737 ms`. You can then access the UI of the Author instance.

### [](#_log_into_the_author_instance)Log into the Author instance

Go to `[http://localhost:8080/magnoliaAuthor](http://localhost:8080/magnoliaAuthor)` and log in to the Author instance with:

-   Username: `superuser`

-   Password: `superuser`


## [](#_installing_the_bookshelf_app)Installing the Bookshelf app

1.  Go to the `dev/light-modules/` directory.

2.  Clone the `bookshelf` Git repository:

    ```sh
    git clone -b release/6.2 --single-branch {repo-git-bookshelf}
    ```

3.  The app should now appear in the Apps **Edit** group.
    ![App Launcher with the Bookshelf icon](../../../../RNs-6-4-3/_images/6-2-29-my1stTrans-Bookshelf-app.png)

4.  In [your user profile](../../IAM/Users/#_editing_user_profile), set the language preference to Spanish. Log out and into Magnolia again. Open the Bookshelf app. Only some of the labels are shown in Spanish. These are typically the more generic message and label keys coming from the UI framework:
    ![Bookshelf app labels shown in Spanish](../../../../RNs-6-4-3/_images/6-0-my1stTrans-browserApp-es_locale-rbox.png)


## [](#_translating_the_app_ui_to_spanish)Translating the app UI to Spanish

In the steps below, you add the missing Spanish translations:

1.  Go to the `dev/light-modules/booshelf/i18n/` subfolder. The `bookshelf-messages_en.properties` file contains the English values for the app labels and messages.

2.  Make a copy of the file and name it `bookshelf-messages_`**`es`**`.properties`.

3.  In the new file, translate the values from English to Spanish and save your work.

    > **Note:** We suggest you translate in an external text editor.If you open the file in the internal editor of the Resource files app, the resource becomes a copy in the JCR workspace that is prioritized by Magnolia over the original file on the file system. If you make subsequent changes to the file on the file system, your changes will not be taken into account. JCR configuration overrides file system configuration.

    Example translation:

    bookshelf-messages\_es.properties
