# Task configuration

Task configuration enables you to define and customize AI-powered tasks in **AI Accelerator**. Tasks represent specific AI operations that can be performed using configured models, such as content generation, image creation, or text analysis.

## [](#_configuration)Configuration

AI tasks can be added to your light module by creating an `aiTasks` folder and adding a `<your-task>.yaml` file with the following configuration:

```yaml
taskName: <your-task-name> (1)
$type: <task-type> (2)
description: <task-description> (3)
model: <model-reference> (4)
input: (5)
  <parameter-name>:
    $type: <PARAMETER-TYPE>
    required: <true/false>
    description: <DESCRIPTION>
    defaultValue: <DEFAULT-VALUE>
output: <6> # This is currently not in use
  <parameter-name>:
    $type: <wd-TYPE>
    description: <DESCRIPTION>
```

|     |     |
| --- | --- |
| **1** | The displayed name of the task in the UI (i.e. "Generate Product Description", "Create Hero Image", etc). |
| **2** | The type of task being configured (i.e. `textToText`, `textToImage`, `imageToImage`). |
| **3** | A description of what the task does, shown in the UI. |
| **4** | Reference to the model configuration that this task will use. |
| **5** | Input parameters that the task requires to execute. |
| **6** | Output parameters that define the structure of the taskâ€™s results. They are used in the [ModelAdapter](../model-configuration/#_model_adapter). |

## [](#_available_tasks)Available tasks

The current version of AI Accelerator supports the following tasks: [`executePromptTask`](#_execute_prompt_task), [`generateComponentTask`](#_component_generation_task) and [`generateVariantTask`](#_variant_generation_task)

### [](#_execute_prompt_task)Execute Prompt Task

```yaml
modelId: gpt-4o
$type: executePromptTask
input:
  prompt:
    $type: prompt
    required: true
    description: The prompt to generate an image from.
  locale:
    $type: string
    required: true
```

### [](#_component_generation_task)Component Generation Task

```yaml
modelId: gpt-4o
$type: generateComponentTask
input:
  prompt:
    $type: prompt
    required: true
    description: The prompt to generate an image from.
  templateId:
    $type: string
    required: true
  locale:
    $type: string
    required: true
```

### [](#_variant_generation_task)Variant Generation Task

```yaml
modelId: gpt-4o
$type: generateVariantTask
input:
  prompt:
    $type: prompt
    required: true
    description: The prompt to generate an image from.
  component:
    $type: object
    required: true
  locale:
    $type: string
    required: true
```

## [](#_parameter_types)Parameter Types

Tasks use the same parameter types as models. See [Parameter Configuration](../parameters/) for detailed information about available parameter types and their configurations.
