# Composite field - 5 UI

Deprecated

> **Note:** This composite field definition has been deprecated since Magnolia 6.0. It is part of the Magnolia 5 UI framework.For the updated implementation, see Composite field for Magnolia 6 UI instead.

`CompositeFieldDefinition` defines a complex field that is composed of multiple simple fields. Use a composite field for objects that cannot be expressed as simple text strings or numbers such as:

-   Setting conditions for selecting the next step in a multistep form.

-   Assigning categories to a content item, where the composite field consists of a category dropdown and a button for reordering categories.

-   Defining an image gallery using a media select field, where the composite media field consist of a thumbnail and caption text.

-   Specifying an access control list for JCR workspaces, where the [composite ACL field](#6.2@product-docs:ROOT:Administration/Security/Roles-and-access-control-lists.adoc) consists of permission, scope and path.


class: `info.magnolia.ui.form.field.definition.CompositeFieldDefinition`

fieldType: `composite`

Here is a composite field that consists of a [text field](../Text-field-5-UI/) and a [date field](../Date-field-5-UI/).

![Composite field](6.2@product-docs:ROOT:composite-field.png)

To export the content of the composite fields for translation via the [Content Translation Support module](#6.2@product-docs:ROOT:Modules/List-of-modules/Content-Translation-Support-module.adoc), the `composite` field type has to be registered in the configuration of the module. For details see [Registering additional field types](#6.2@product-docs:ROOT:Modules/List-of-modules/Content-Translation-Support-module.adoc#_registering_additional_field_types).

## [](#_composite_field_properties)Composite field properties

Simple composite field definition:

```js
form:
  tabs:
    - name: tabEvent
      label: Event
      fields:
        - name: title
          fieldType: composite
          label: Event
          fields:
            - name: title
              fieldType: text
              label: Title
            - name: date
              fieldType: date
              label: Date
```
