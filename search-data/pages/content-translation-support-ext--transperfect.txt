# CTSX Transperfect

[Content management](../../product-docs/7.0/use-cases/content-management/) [Incubator](https://gitlab.magnolia-platform.com/pd/extensions/ps-maintained/content-translation-support-ext) Version 5.0.0 [Magnolia 6.4](../../product-docs/7.0/) compatible

The **CTSX Transperfect** module performs translations using the Transperfect APIs. Translations are handled asynchronously.

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../product-docs/7.0/Developing/Bundles-and-webapps/):

> **Important:** In order to start the translation process, please make sure that the core and apps modules in the CTSX Framework have already been installed.

```xml
<dependency>
  <groupId>info.magnolia.translation</groupId>
  <artifactId>magnolia-content-translation-support-ext-translations-com</artifactId>
  <version>5.0.0</version>
</dependency>
```

```xml
<dependency>
  <groupId>com.translations.globallink</groupId>
  <artifactId>globallink-connect-api</artifactId>
  <version>4.18.4</version>
</dependency>
```

## [](#_configuration)Configuration

The configuration can be done using JCR or YAML.

-   Refer to the [JCR Configuration](#_jcr_configuration) for the JCR configuration.

-   Refer to the [YAML configuration](#_yaml_configuration) for instructions on how to define settings using YAML.


### [](#_jcr_configuration)JCR Configuration

Configuration of the Transperfect service provider is handled in the module config. This provider should be configured with the following path:

```none
modules
└── content-translation-support-ext-translations-com
    └── config
        └── translationProviders
            └── transComProviderConfigFile.yaml
```

> **Note:** For users upgrading from version 4.0.0 to version 4.1.0, ensure you move your existing translation provider configuration to the correct path.

### [](#_yaml_configuration)YAML configuration

Transperfect service provider configuration is defined within its provider module (`magnolia-content-translation-support-ext-microsoft`) under the following directory structure:

```plaintext
/src/main/resources/magnolia-content-translation-support-ext-translations-com/translationProviders
```

#### [](#_required_decorations)Required decorations

This configuration can also be **decorated** to override or extend the default configuration. Decoration file should be placed under the following path:

```plaintext
<light-module-name>/decorations/magnolia-content-translation-support-ext-translations-com/translationProviders
```

Below is an example of how to decorate Transperfect service provider configuration within a light module:

```plaintext
light-modules/
└── my-module/
    └── decorations/
        └── magnolia-content-translation-support-ext-translations-com/
            └── translationProviders/
                └── Transperfect.yaml
```

### [](#_configuration_properties)Configuration properties

This provider can be configured with the following properties.

| Property | Description |
| --- | --- |
| `agentName` | **optional**<br><br>Translation agent name. |
| `autoTranslationAllowed` | **optional** , default is `false`<br><br>When true the content review process (workflow) is bypassed. |
| `class` | **required**<br><br>Definition class for the translator to be used.<br><br>See `info.magnolia.translation.ext.provider.google.definition.TranslationsComProviderDefinition`. |
| `clientId` | **required**<br><br>Identifies the client application. |
| `clientPassword` | **required**<br><br>Identifies the client password. |
| `configName` | **required**<br><br>Identifies the configuration within Magnolia. |
| `daysListen` | **required**<br><br>Number of days system continue to listen to GlobalLink PD (calculated from last modified date of a submission). |
| `defaultFlag` | **optional**<br><br>The default configuration used by the system in the case of multiple translation services being used. |
| `enabledFlag` | **required**<br><br>Enable or disable the configuration using this flag. |
| `fileFormat` | **required**<br><br>Depends on the version of GlobalLink PD you use. For version 5 use "Magnolia". For version 4 use "xliff". |
| `implementationClass` | **required**<br><br>Implementation class for interfacing with Transperfect.<br><br>See `info.magnolia.translation.ext.provider.translationscom.translator.TranslationsComRestTranslator`. |
| `projectId` | **only required in 3.4.10+**<br><br>Id of the project setup with Transperfect. |
| `basicKey` | **only required in 3.4.10+**<br><br>This basic auth token is only used to get PD credential. You can get these credentials from a PD administrator. |
| `submissionPrefix` | **optional**<br><br>Prefix added to submitted documents.<br><br>Example: `magnolia-` |
| `downloadExpiration` | **optional, default is 840000**<br><br>Download expiration time in milliseconds. Default to 14 minutes (840,000 ms). |
| `headerParams` | **required**<br><br>Additional request headers to be sent on submission creation, configured as a content node of properties where the property key is the header name, and the property value is the header value.<br><br>The `User-Agent` header is required and is set in the following format:<br><br>`<connectorName> <connectorVersion>_<buildtype> (<clientVersion>)`<br><br>`buildtype` is only required if release is not final: Alpha, Milestone, RC, etc. (These cannot be deployed to production)<br><br>`clientVersion` is optional and refers to the name/version of the system the connector is built for. This information is useful for troubleshooting.<br><br>Example: `Magnolia 4.1.1 PROD` |
| `techTracking` | **required**<br><br>Tech tracking is a set of four key/value pairs that allow an integration to specify the following information:<br><br>**adaptorName** - Name of the adaptor that created the submission  <br><br>**adaptorVersion** - Version of the adaptor  <br><br>**clientVersion** - Name and version of the client system  <br><br>**technologyProduct** - Code of the technology product that created the submission; please check with the GL Support team for the code of your specific technology product |

```yaml
      'clientId': 'xxx'
      'clientPassword': 'xxx'
      'configName': 'translations.com'
      'daysListen': '1'
      'defaultFlag': 'false'
      'enabledFlag': 'true'
      'fileFormat': 'xxx'
      'implementationClass': 'info.magnolia.translation.ext.provider.translationscom.translator.TranslationsComRestTranslator'
      'projectId': 'xxx'
      'basicKey': 'xxx'
      'submissionPrefix': 'xxx'
      'url': 'xxx'
      'headerParams':
        'User-Agent': 'Magnolia 4.1.1 PROD'
      'techTracking':
        'adaptorName': 'xxx'
        'adaptorVersion': 'xxx'
        'clientVersion': 'xxx'
        'technologyProduct': 'xxx'
```

## [](#_usage)Usage

After registration and starting, the Content Translation Support Extended (CTSX) module will bind the Transperfect translator to `Configuration` field.

> **Note:** You can test the connection to the translator api before using.

![translations dot com](_images/translations-dot-com.jpg)

### [](#_how_it_works)How it works

Use the same steps outlined with [CTSX Google Translator](../google/) but you will choose **Transperfect** instead of Google.com in the Configuration.

> **Important:** In additional, at the Step 2 after submitting the batch you must to wait for the response from Transperfect’s GlobalLink Project Director app (PD).

> **Note:** You would be able to see the Pulse notification for translation result as soon as the submission has been delivered on PD.

-   5.x

-   4.x
