# Rich text field - 5 UI

Deprecated

> **Note:** This rich text field definition has been deprecated since Magnolia 6.0. It is part of the Magnolia 5 UI framework.For the updated implementation, see Rich text field for Magnolia 6 UI instead.

`RichTextFieldDefinition` renders a rich text editor. This is a [custom Magnolia field](../../Custom-fields-5-UI/) that implements the [CKEditor](http://docs.ckeditor.com/) component.

class: `info.magnolia.ui.form.field.definition.RichTextFieldDefinition`

fieldType: `richText`

![Text form with example text](6.2@product-docs:ROOT:6-0-textAndImage.png)

## [](#_rich_text_field_properties)Rich Text field properties

Simple rich text field definition:

```js
form:
  tabs:
    - name: tabText
      label: Text
      fields:
        - name: text
          fieldType: richText
          tables: true
          height: 500
          label: Text Editor
```

## [](#_loading_order_for_custom_configuration)Loading order for custom configuration

> **Warning:** Any configuration done in RichTextFieldFactory will override configuration settings from the config.js file. This is why we ignore other properties in the field definition as soon as a configJsFile is configured.

## [](#_decode_text)Decode text

CKEditor produces HTML such as `<p>` elements for paragraphs. However, text stored in JCR usually escapes HTML code. To render text which originates from a rich text field you need to decode the stored content to make sure the HTML tags are rendered properly again.

Decode the `content` object and then get the desired property from it:

```none
${cmsfn.decode(content).text!""}
```

See [`cmsfn` decodeHtml](#6.2@product-docs:ROOT:Developing/Templating/Template-scripts/Templating-functions/cmsfn.adoc#_decode_html).

## [](#_impact_on_the_search_index)Impact on the search index

Keep in mind that using the RTE introduces HTML into your content. The impact of this is that the HTML will be indexed along with the content.
