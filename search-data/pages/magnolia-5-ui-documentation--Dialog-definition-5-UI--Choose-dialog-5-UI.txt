# Choose dialog - 5 UI

Deprecated

> **Note:** Choose dialog has been deprecated since Magnolia 6.0. It is part of the Magnolia 5 UI framework.For the updated implementation, see Chooser dialog for _Magnolia 6 UI instead.

A choose dialog allows you to select an item from an app, typically from a [content app](#6.2@product-docs:ROOT:Apps/Developing-an-app/My-first-content-app.adoc). Below is a choose dialog for choosing a page. The dialog displays the workbench from the Pages app. You typically open a choose dialog by clicking a [link field](../Field-definition-5-UI/List-of-fields-5-UI/Link-field-5-UI/) in some other dialog. Configure the choose dialog under the app that manages the items.

![Pages chooser](6.2@product-docs:ROOT:pages-chooseDialog.png)

## [](#_choosing_items_from_content_apps)Choosing items from content apps

A content app already has a suitable view for choosing items: the workbench of the first subapp. You don’t need to configure a choose dialog explicitly. `AppContext` creates `ChooseDialogDefinition` programmatically. `WorkbenchFieldDefinition` and `ContentConnectorDefinition` are instantiated in the `ContentApp#openChooseDialog` method. Values for the definition classes are cloned from the first subapp of the app.

![Dialog launching diagram](6.2@product-docs:ROOT:ChooseDialog-launching-diagram.png)

## [](#_configuring_a_choose_dialog)Configuring a choose dialog

> **Tip:** You don’t need to configure a choose dialog when choosing items from a content app. The workbench of the first subapp works as an item chooser automatically.

You can configure a choose dialog explicitly if required. Create a `chooseDialog` node under the app configuration.

![Choose dialog configuration](6.2@product-docs:ROOT:ADVT-choose-dialog.png)

| Property | Description |
| --- | --- |
| `chooseDialog` | **optional**<br><br>Choose dialog node. |
| `contentConnector` | **required**<br><br>Content connector node. |
| `class` | **required**<br><br>[Content connector definition class](../../Content-apps-5-UI/App-configuration-5-UI/Content-connector-5-UI/) implementing `info.magnolia.ui.vaadin.integration.contentconnector.ContentConnectorDefinition`. |
| `field` | **required**<br><br>Field node. |
| `class` | **required**<br><br>A field used to choose the item, typically a [workbench](../../Content-apps-5-UI/App-configuration-5-UI/Workbench-definition-5-UI/). The value must be a subclass of `info.magnolia.ui.form.field.definition.FieldDefinition`. It can be any kind of field (for instance, `SelectFieldDefinition` or a more complex custom field). |
| `class` | **required**<br><br>A choose dialog definition class implementing `info.magnolia.ui.dialog.definition.ChooseDialogDefinition`.  <br>Its default implementation is `info.magnolia.ui.dialog.definition.ConfiguredChooseDialogDefinition` (due to registration magnolia-ui-framework/src/main/resources/META-INF/magnolia/ui-framework.xml). Leave out the `class` property to use the default implementation. |

> **Note:** ChooseDialogDefinition extends DialogDefinition. This means you can also configure actions (Select and Cancel by default) and an actionArea (optional).

### [](#_configuring_a_workbench)Configuring a workbench

If you want to configure a custom workbench, use `info.magnolia.ui.contentapp.field.WorkbenchFieldDefinition` as field.

![Workbench configuration](6.2@product-docs:ROOT:ADVT-workbench.png)

| Property | Description |
| --- | --- |
| `chooseDialog` | **optional**<br><br>Choose dialog node. |
| `field` | **required**<br><br>Field node. |
| `workbench` | **optional**<br><br>Extend an existing workbench definition or use any properties from the [workbench definition](../../Content-apps-5-UI/App-configuration-5-UI/Workbench-definition-5-UI/). |
| `extends` | **optional**<br><br>Path to extended definition. |
| `imageProvider` | **optional**<br><br>[Image provider definition](../../Content-apps-5-UI/App-configuration-5-UI/Image-provider-definition-5-UI/). |
