# Column definition

Columns define what content is displayed in tree and list views in the browser subapp.

> **Note:** This column definition is part of the Magnolia 6 UI framework. The fully qualified class name is info.magnolia.ui.contentapp.configuration.column.ColumnDefinition.

## [](#_example_definitions)Example definitions

### [](#_simple_column_definitions)Simple column definitions

A `componentColumn` can render different components based on the properties of the item in each row. A common use case is to display a different icon for each node type in a tree view.

In the Title column example below, the `nodeTypeToComponents` property maps node types to specific, inline component configurations. In particular, the block of properties nested under `mgnl:page`. For each row, the column checks the item’s node type. If the node type is `mgnl:page`, this configuration applies, meaning only rows for pages display the `icon-file-webpage` icon and show the node’s path. If a row’s item has a different node type, this specific configuration isn’t applied.

-   Title column

-   Date column


```yml
columns:
  jcrName:
    $type: componentColumn
    nodeTypeToComponents:
      mgnl:page:
        icon: icon-file-webpage
        showPath: true
```

```yml
columns:
  mgnl:lastModified:
    $type: dateColumn
```

### [](#_columns_with_filter_definitions)Columns with filter definitions

-   With a default filter value

-   With a checkbox filter


```yml
columns:
  value:
    filterComponent:
      $type: textField
      defaultValue: '10'
      converterClass: info.magnolia.ui.editor.converter.StartsWithFilteringModeConverter`
```

```yml
columns:
  mgnl:activationStatus:
    $type: columnComponent
    type: java.lang.Boolean
    filterComponent:
      $type: checkBoxField
      defaultValue: true
```

### [](#_columns_with_combobox_filters_and_datasource_definitions)Columns with combobox filters and datasource definitions

-   With an enumeration

-   With options

-   With a list of JCR nodes


```yml
columns:
  layout:
    filterComponent:
      $type: comboBoxField
      emptySelectionAllowed: true
      datasource:
        $type: enumDatasource
        enumeration: info.magnolia.ui.field.Layout
```

```yml
columns:
  isFeatured:
    filterComponent:
      $type: comboBoxField
      emptySelectionAllowed:
      datasource:
        $type: optionListDat
        options:
          - name: true
            value: true
            label: Featured
```

```yml
columns:
  destination:
    filterComponent:
      $type: comboBoxField
      emptySelectionAllowed: tr
      datasource:
        $type: jcrDatasource
        rootPath: /destinations
        workspace: category
```

## [](#_column_properties)Column properties

 PropertyDescription

`class`

**required** (unless `$type` is used)

The column definition class reads the column configuration and displays the column accordingly. The class must implement the info.magnolia.ui.contentapp.configuration.column.ColumnDefinition interface. You can write your own class or use one of the ready-made classes. See [Column types](#_column_types) for possible values.

`$type`

You can use this as a shortcut for `class` if the definition class is annotated with `info.magnolia.ui.contentapp.configuration.column.ColumnType`. The proper value is defined by the annotation.

Example class annotation

```java
@ColumnType("dateColumn")
public class DateColumnDefinition extends ConfiguredColumnDefinition<Date> {
...
}
```

To use the `$type` property in YAML, see [Example definitions](#example_definitions).

`type`

**required**, *default is `java.lang.String`*

Data type of the column. Use the fully qualified class name. See [PropertyType](https://developer.adobe.com/experience-manager/reference-materials/spec/jsr170/javadocs/jcr-2.0/javax/jcr/PropertyType.html) for possible values.

A default type is typically hard-coded in each definition class. You only need to add this property if you want to override the default implementation.

> **Important:** Make sure that type is set to a value that matches the type of data you entered (for example, java.lang.Long for long values).

> **Caution:** You can’t configure this property through YAML but it can be used in your Java code.

`name`

**optional**, *default is parent node name*

Name of the column.

`label`

**optional**

Text displayed in the column heading. The value can be literal or a [key](../../../Administration/Language/Internationalization/#_i18n_message_keys) of a [message bundle](../../../Administration/Language/Internationalization/#_i18n_message_bundle).

If you don’t provide the property, Magnolia falls back to a generated i18n key.

`descriptionGenerator`

**optional**

Generates a description for the column.

> **Caution:** You can’t configure this property through YAML but it can be used in your Java code.

`editable`

**optional**, *default is `false`*

When `true`, the column can be edited inline. You can double-click a cell to edit its value.

> **Warning:** You can’t define defaultAction and use inline editing at the same time.

`editor`

**optional**

Defines the inline editor component. For this to work, you have to set `editable` to `true`. See the info.magnolia.ui.contentapp.configuration.column.ColumnEditorDefinition interface.

     `field`

**required**, *default is `textField`*

Type of the field definition item. See [Field types](../../../Developing/Templating/Dialog-definition/Field-definition/#_field_types) for possible values.

     `availability`

**optional**

Outlines when inline editing is permitted. See [Action availability](../Action-definition/#_action_availability) for more information.

> **Important:** Availability is resolved against the selected row, not the selected column or property.

`filterComponent`

**optional**

Adds a filter field to the column. Only [text](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Text-field/), [checkbox](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Checkbox-field/) and [combobox](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Combobox-field/) fields are supported (see [Example definitions](#example_definitions)). The default value and converter of [combobox field](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Combobox-field/) are not supported.

A node identifier is applied to filtering if the field value is `javax.jcr.Node`. That is, if the `destination` column in [Example definitions](#example_definitions) is configured with [`jcrDatasource`](../Data-source-definition/JCR-data-source/), the [combobox field](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Combobox-field/) options will be populated with `javax.jcr.Node`.

> **Tip:** Use converterClass: info.magnolia.ui.editor.converter.StartsWithFilteringModeConverter to change the filtering mode from CONTAINS to STARTSWITH to improve the performance of textField filters in huge JCR workspaces.

> **Note:** Column filtering is not supported in REST-based elements of the UI.

`expandRatio`

**optional**

Defines the ratio with which the column expands. By default, all columns expand equally.

`maximumWidth`

**optional**

Defines the maximum allowed pixel width of the column when it is set to expand.

`minimumWidth`

**optional**

Defines the minimum guaranteed pixel width of the column when it is set to expand.

`minimumWidthFromContent`

**optional**, *default is `false`*

Sets whether the width of the content in the column is the minimum width of the column. When `false`, the column shrinks down to the width defined in `minimumWidth` if necessary.

`width`

**optional**

Defines the width (in pixels). When set, overrides any configuration from `expandRatio`, `maximumWidth` and `minimumWidth`.

`sortable`

**optional**, *default is `true`*

When `false`, the column cannot be sorted.

`renderer`

**optional**, *default is `com.vaadin.ui.renderers.TextRenderer`*

Class extending Vaadin [`AbstractRenderer`](https://vaadin.com/api/framework/8/com/vaadin/ui/renderers/AbstractRenderer.html).

*com.vaadin.ui.renderers.ComponentRenderer* is used by *columnComponent*. *info.magnolia.ui.contentapp.configuration.column.renderer.DefaultDateRenderer* is used by *dateColumn*.

> **Caution:** You can’t configure this property through YAML but it can be used in your Java code.

`valueProvider`

**optional**

Class extending Vaadin [`ValueProvider`](https://vaadin.com/api/framework/8/com/vaadin/data/ValueProvider.html).

> **Caution:** You can’t configure this property through YAML but it can be used in your Java code.

> **Tip:** We don’t recommend configuring type, renderer and valueProvider through YAML.Rather, create a custom column definition instead to prevent misconfiguration errors such as configuring a correct renderer but a wrong type:public class ProgressColumnDefinition<T> extends ConfiguredColumnDefinition {
>     public ProgressColumnDefinition() {
>         setType(Double.class);
>         setRenderer(ProgressBarRenderer.class);
>     }
> }columns:
>   scale:
>     class: my.package.ProgressColumnDefinition
