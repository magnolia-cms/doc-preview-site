# Admincentral module

The Admincentral module is used to configure the layout and availability of apps in the [App launcher](../../../Administration/App-Launcher/App-launcher-layout/).

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../Developing/Bundles-and-webapps/):

```xml
<dependency>
  <groupId>info.magnolia.admincentral</groupId>
  <artifactId>magnolia-admincentral</artifactId>
  <version>6.4.0</version> (1)
</dependency>
```

|     |     |
| --- | --- |

Properties to configure the App launcher layout.

| Property | Description |
| --- | --- |
| `favouriteAppConfiguration` | Configuration for favorite applications.<br><br>Example:<br><br>```yaml<br>favouriteAppConfiguration:<br>  maxApps: 12<br>  favouriteApps:<br>    - pages-app<br>``` |
| `favouriteApps` | **optional** (type: `List`)<br><br>List of favorite application names. |
| `maxApps` | **optional** (type: `int`)<br><br>Maximum number of favorite apps allowed. |
| `layout` | Configuration for layout settings. |
| `defaultGroup` | **optional** (type: `String`)<br><br>Default group name for apps. |
| `hiddenApps` | **optional** (type: `List`)<br><br>List of apps to hide. |
| `groups` | **optional** (type: `List`)<br><br>List of group configurations.<br><br>Example:<br><br>```yaml<br>layout:<br>  groups:<br>    - name: content<br>      apps:<br>        - name: pages-app<br>        - name: dam<br>    - name: marketing<br>      apps:<br>        - name: analytics<br>        - name: campaigns<br>    - name: admin<br>      apps:<br>        - name: security-app<br>      permissions:<br>        groups:<br>          - developers<br>``` |
| `name` | **required** (type: `String`)<br><br>Name of the group. |
| `label` | **optional** (type: `String`)<br><br>Display label for the group. |
| `apps` | **optional** (type: `List`)<br><br>List of app configurations within the group. |
| `name` | **required** (type: `String`)<br><br>Name of the app within the group. |
| `permissions` | **optional** (type: `Object`)<br><br>Permission settings for the group. |
| `groups` | **optional** (type: `List`)<br><br>List of groups with access permissions. |
| `rows` | **optional** (type: `List`)<br><br>List of row configurations.<br><br>Example:<br><br>```yaml<br>laayout:<br>  rows:<br>    - name: one<br>      cssClass: editor<br>      groups:<br>        content: content<br>        marketing: marketing<br>    - name: two<br>      cssClass: editor<br>      groups:<br>        - personalisation<br>``` |
| `name` | **required** (type: `String`)<br><br>Name of the row. |
| `displayed` | **optional** (type: `boolean`)<br><br>Indicates whether the row is displayed (default: `true`). |
| `cssClass` | **optional** (type: `String`)<br><br>CSS class to apply to the row. |
| `groups` | **optional** (type: `List` or `Map`)<br><br>List or map of groups to include in the row. |

## [](#_global_password_policy_validation)Global password policy validation

The Admincentral module provides global password policy validation for all password fields via the ValidatingPasswordFieldDefinition class. This validation is configured in the `ui-framework-core` module and enforces password strength requirements across the system.

The default password policy requires:

-   Minimum 12 characters in length

-   At least one uppercase English letter

-   At least one lowercase English letter

-   At least one digit

-   At least one special character from the set: `#?!@$%^&*-`


This policy is automatically applied to password fields in the [Security app](../../../Apps/List-of-apps/Security-app/) and any other location where password validation is needed.

### [](#_configure_the_password_policy_pattern)Configure the password policy pattern

You can customize the password policy by decorating the `ui-framework-core` module configuration. The `pattern` property uses a regular expression to define password requirements.

To modify the password policy, create a decoration file in your module:

ui-framework-core/config.yaml

```yaml
...
  info.magnolia.ui.field.ValidatingPasswordFieldDefinition$BcryptedPasswordFieldDefinition:
    policy:
      class: info.magnolia.ui.field.RegexpValidatorDefinition
      # Password policy regular expression below:
      ## Password has a minimum of 12 characters in length. Adjust it by modifying {12,}
      ## At least one uppercase English letter. You can remove this condition by removing (?=.*?[A-Z])
      ## At least one lowercase English letter. You can remove this condition by removing (?=.*?[a-z])
      ## At least one digit. You can remove this condition by removing (?=.*?[0-9])
      ## At least one special character. You can remove this condition by removing (?=.*?[#?!@$%^&*-])
      pattern: ^(?=.*?[A-Z])(?=.*?[a-z])(?=.*?[0-9])(?=.*?[#?!@$%^&*-]).{12,}$
```

You can customize the pattern by:

-   Changing the minimum length by modifying `{12,}` to your desired number

-   Removing character type requirements by deleting the corresponding `(?=.*?[…​])` groups

-   Adding additional special characters to the character set `[#?!@$%^&*-]`


For more details about password field configuration, see [Password field](../../../Developing/Templating/Dialog-definition/Field-definition/List-of-fields/Password-field/).
