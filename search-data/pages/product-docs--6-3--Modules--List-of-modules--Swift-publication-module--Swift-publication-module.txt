# Swift publication module

| **Edition** | DX Core |
| --- | --- |
| **License** | [MLA](https://www.magnolia-cms.com/mla.html) |
| **Issues** | [SWIFTPUB](https://magnolia-cms.atlassian.net/browse/SWIFTPUB) |
| **Maven site** | [Swift publication](https://nexus.magnolia-cms.com/#browse/browse:enterprise:info%2Fmagnolia%2Fswift%2Fpublication%2Fmagnolia-swift-publication%2F2.0.0-beta-1) |
| **Latest** | 2.0.0-beta-1 |

| ArtifactID | Description |
| --- | --- |
| `magnolia-swift-versioning-api` | Provides an API to create and restore versions in an external version store. |
| `magnolia-swift-versioning-ui` | Includes the integration of the user interface to manage versions in a module. Provides a means to manage versions in a module, including implementations for version checks and restoration. |
| `magnolia-swift-versioning-s3` | Provides functionality to implement S3 version storage. |
| `magnolia-swift-publication-ui` | Provides decorations for the Pages app, publishing dialogs, and i18n labels for the new UI buttons. |
| `magnolia-swift-publication-api` | Provides an API for external publishing, including services to process publication requests sent by authors, to represent problems that occur when a parent node is not yet published, and the ordering strategy during publication. |
| `magnolia-swift-publication-rest` | Provides a REST publication client with the following. • A REST publication endpoint • Publishing and unpublishing operations • Exception handling adhering to the problem details for HTTP APIs specification as defined by RFC 7807 • Ensuring proper serialization and deserialization • Dealing with invalid API tokens |
| `magnolia-swift-publication-personalization` | Extends `PublicationCommand` and handles deletion for [personalization](#Features/Personalization) when page variants are deleted. |
| `magnolia-swift-versioning-diff` | Integrates the [Diff module](../../Diff-module/) functionality, allowing you to use it with Swift publication. |

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../../../RNs-6-4-3/Developing/Bundles-and-webapps/):

Some steps below are required, and some parts are optional.

1.  Add the following mandatory dependencies.

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-api</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-ui</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-api</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-ui</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-diff</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

2.  Add the S3 storage service dependency.

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-s3</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

3.  Add REST communication between the author and the public instances dependency.

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-rest</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

4.  If required, add a dependency for the Personalization module.

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-personalization</artifactId>
      <version>2.0.0-beta-1</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | It’s good practice to use the `version` variable from the parent POM (see in the last step), hence `{mgnlSwiftVersion}` would be here instead of `2.0.0-beta-1`. |

5.  Add the module version to the `properties` of the parent POM.

    Parent POM

    ```xml
    <properties>
      …​
      <mgnlSwiftVersion>2.0.0-beta-1</mgnlSwiftVersion>
      …​
    </properties>
    ```

    > **Important:** Update dependency management in your project.
