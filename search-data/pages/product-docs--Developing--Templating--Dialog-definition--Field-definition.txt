# Field definition

A field is an HTML form element that allows you to perform a task such as enter text, upload an image or select a date.

> **Note:** This field definition is part of the Magnolia 6 UI framework. The fully qualified class name is info.magnolia.ui.field.FieldDefinition.

## [](#_field_examples_module)Field examples module

You can download an example light module called [field-examples-module](../../../../_attachments/field-examples-module.zip) . The module contains working field definitions to use out of the box for all fields in this section. Follow the instructions below to preview them in your local Magnolia installation.

1.  Set the content bootstrap folder, pattern, and strategy in the `WEB-INF/config/default/magnolia.properties file`. For example:

    ```none
    magnolia.content.bootstrap.dir=/<LIGHT-MODULE-FOLDER-LOCATION>/field-examples-module/bootstrap/data
    magnolia.content.bootstrap.pattern=*.{yaml,xml}
    magnolia.content.bootstrap.createTasks=always
    ```

2.  Unzip and copy the [fields-examples-module](../../../../_attachments/field-examples-module.zip) into your light-modules folder.

    > **Note:** Two field examples use the books workspace. Previewing those fields requires completing the Bookshelf app tutorial or cloning the bookshelf repository directly into your light-modules folder.

3.  Start up your Magnolia instance and log in.

4.  You should see three import tasks in the **Tasks** app. One is for an asset, one to import books, and another for a page. Import all files.

5.  Go to the `Pages` app and check that the `Field examples` page is successfully bootstrapped.

6.  Edit the new page and add a new component. This opens a dialog showing all the component types available.

    ![Add component dialog](../../../../_images/field-examples_component-types.png)

7.  After selecting a component type and clicking `Next`, the respective dialog fields are shown.

    > **Tip:** You can change the values, save them, and then edit them again later.


## [](#_example_definition)Example definition

To add a field to your [form](../Form-definition/), you need to define a field. Field definition must contain a [`class`](#_field_types) or [`$type`](#_field_types) property, which tells Magnolia what type of field to render. Set the property value to a fully qualified class name (for `class`) or field alias (for `$type`). Depending on the field type, a number of common [simple field properties](#_common_simple_field_properties) or [complex field properties](#_common_complex_field_properties) are supported.

```yml
firstName:
  label: First name
  $type: textField
  required: true
lastName:
  label: Last name
  $type: textField
  required: true
email:
  label: Email
  $type: textField
  required: true
  validators:
    email:
      $type: emailValidator
```

## [](#_i18n_in_new_ui_forms_and_vaadin_forms)i18n in New UI Forms and Vaadin forms

The rendering and behavior of internationalization (i18n) differ between the fields of the [New UI Forms](../New-ui-forms/) and the older Vaadin forms.

### [](#_fields_with_i18n_true)Fields with `i18n: true`

| New UI Forms | Vaadin |
| --- | --- |
| No language identifier is displayed next to the field’s name.<br><br>The field value you edit there and save is stored only for the selected language variant. | A two-letter language identifier (for example, `en`, `de`, `fr`) is displayed next to the field, as shown below on the right side of the screenshots - `en` or `de` next to the "Body Text" name of the rich text field.<br><br>The field value you edit there and save is stored only for the selected language variant. |

### [](#_fields_with_i18n_false_or_not_set_as_false_is_the_default)Fields with `i18n: false` (or not set, as `false` is the default)

The field is disabled in non-default language views.</p></div><div class="paragraph"><p>A globe icon is displayed next to the field.</p></div><div class="paragraph"><p>Hovering over the globe icon shows a tooltip with the following messages:</p></div><div class="ulist"><ul><li><p>In the default language view: "Global field.
