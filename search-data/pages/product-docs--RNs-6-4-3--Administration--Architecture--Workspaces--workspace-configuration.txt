# Workspace configuration

Once a workspace is created, you can adjust the workspace configuration on a workspace-by-workspace basis. For each new workspace, there is a corresponding `workspace.xml` file for fine-tuning individual performance. By default, the file is located in the file system inside the corresponding workspace folder.

> **Note:** To modify the configuration of an existing workspace, you need to change the workspace.xml file for that workspace. Changing the <Workspace/> element in the repository configuration file does not affect existing workspaces.

## [](#_file_system)File system

The virtual file system passed to the persistence manager and search index.

```xml
<FileSystem class="org.apache.jackrabbit.core.fs.local.LocalFileSystem">
  <param name="path" value="${rep.home}/repository" />
</FileSystem>
```

Jackrabbit provides a lot of choices for how you can configure the `FileSystem`. Choose the class that best fits your use case.

-   [local](https://jackrabbit.apache.org/api/2.20/org/apache/jackrabbit/core/fs/local/LocalFileSystem.html)

-   [db](https://jackrabbit.apache.org/api/2.20/org/apache/jackrabbit/core/fs/db/package-summary.html)

-   [in-mem](https://jackrabbit.apache.org/api/2.20/org/apache/jackrabbit/core/fs/mem/MemoryFileSystem.html)


## [](#_persistence_manager)Persistence manager

Each workspace in a Jackrabbit content repository uses separate persistence managers to store the content in that workspace.

```xml
<PersistenceManager class="org.apache.jackrabbit.core.persistence.pool.DerbyPersistenceManager">
  <param name="url" value="jdbc:derby:${wsp.home}/db;create=true"/>
  <param name="schemaObjectPrefix" value="${wsp.name}_"/>
</PersistenceManager>
```

Jackrabbit provides a lot of choices for how you can configure the `PersistenceManager`. Choose the class that best fits your use case.

-   [pool](https://jackrabbit.apache.org/api/2.20/org/apache/jackrabbit/core/persistence/pool/package-summary.html)

-   [in-mem](https://jackrabbit.apache.org/api/2.20/org/apache/jackrabbit/core/persistence/mem/InMemPersistenceManager.html)

## [](#_disabling_indexing_for_the_mgnlversion_workspace_manually)Disabling indexing for the `mgnlVersion` workspace manually

When upgrading, you can manually disable indexing for the `mgnlVersion` workspace.

1.  Stop the instance.

2.  Remove the `SearchIndex` tag and its contents from `repositories/magnolia/workspaces/mgnlVersion/workspace.xml`.

3.  Delete the entire `repositories/magnolia/workspaces/mgnlVersion/index` folder together with its contents.

4.  Restart the instance.
