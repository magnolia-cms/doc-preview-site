# Recovering the Live environment

| Type | Description |
| --- | --- |
| Restoring from backup | This recovers: • Magnolia software **custom configuration** ( *JCR data of configuration workspace* ). • Editor-created **content** ( *JCR data from content workspaces such as website, dam, category, contacts, etc.* ). |
| [Promoting a release to Live](../../../managing-updates-using-the-Magnolia-cockpit/promoting-a-release-to-Live/) | This recovers **light modules** that you have added yourself. |

## [](#_restore_from_backup)Restore from backup

1.  In the Package overview page click **Manage environments** in the Environments section.

2.  Under **Back up and Restore**, click **Restore Content**.
    ![user interface restore environment](../../../_images/cloud/replace-with-backup.png)

3.  Select the backup from which you want to restore content and configuration.

    > **Note:** Choose either the latest Manual backup or one of the Automated backups. You are informed if there are differences between the backup you have selected and your current Live environment.

4.  Acknowledge your actions and click **Restore**.

5.  A notification banner is displayed and the restore operation appears in the [activity log](../../../Cockpit/Understanding-activity-logs/).
