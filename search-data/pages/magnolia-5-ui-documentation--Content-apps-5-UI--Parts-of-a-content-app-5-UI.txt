# Parts of a content app - 5 UI

Deprecated

> **Note:** This content app framework has been deprecated since Magnolia 6.0. It is part of the Magnolia 5 UI framework.

The content app user interface consists of [subapps](../App-configuration-5-UI/Subapp-descriptor-5-UI/), a [workbench](../App-configuration-5-UI/Workbench-definition-5-UI/) and an [action bar](#6.2@product-docs:ROOT:Apps/App-configuration/Action-bar-definition.adoc). These reusable parts let you build content apps quickly.

## [](#_subapps)Subapps

A content app always has at least two subapps: browser and detail. A subapp is typically displayed to users as a tab.

### [](#_browser_subapp)Browser subapp

A browser subapp gives you a workbench that displays the content of a workspace. It can display the content in tree, list and thumbnail views. It also provides an action bar with action for managing the content items.

![Browser subapp](6.2@product-docs:ROOT:contacts-browser.png)

### [](#_detail_subapp)Detail subapp

A detail subapp provides an [editor](#_editor) for editing the items.

![Detail subapp](6.2@product-docs:ROOT:contacts-detail.png)
