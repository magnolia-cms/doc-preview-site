# Clean up scripts using Groovy

Magnolia Groovy module adds Groovy capabilities to Magnolia. Groovy is a popular dynamic language for the JVM. To know more about it, visit the [Groovy official website](http://www.groovy-lang.org/) which has many tutorials and documentation both for beginners and advanced users of the language.

The module provides:

-   A web based Unix-like console where you can access contents in Magnolia repositories in a `groovyish` way

-   A `scripts` repository where you can store your scripts

-   The ability to plug in Groovy classes into Magnolia at runtime, without the need for deploying them and restarting the servlet container.


All these tools make for a more agile approach to coding and maintaining Magnolia-based websites.

For more information, see [Groovy module](../../../product-docs/RNs-6-4-3/Modules/List-of-modules/Groovy-module/).

## [](#_archive_resolved_tasks)Archive resolved tasks

Resolving tasks is important because removing or archiving tasks that aren’t resolved can result in orphaned workflows. The script below doesn’t cause orphaned workflows because it operates on resolved tasks only.

-   Sample script

-   Console output


[archive-resolved-tasks.groovy](https://bitbucket.org/magnolia-cms/groovy-shell-scripts/src/master/examples/archive-resolved-tasks.groovy)

```none
archived 10 tasks
```

> **Tip:** Alternatively, use the Tasks Cleaner module instead of the Groovy script above.

## [](#_clear_notifications)Clear notifications

Clear all the notifications for a given user.

-   Sample script

-   Console output


[clear-notifications-for-user.groovy](https://bitbucket.org/magnolia-cms/groovy-shell-scripts/src/master/examples/clear-notifications-for-user.groovy)

Example of usage

```bash
clearNotificationsForUser('superuser'); (1)
```

|     |     |
| --- | --- |
| **1** | Notifications removed for `superuser`. |

```none
archived 7 notifications
```

## [](#_remove_messages)Remove messages

Removes all messages from the specified user.

-   Sample script

-   Console output


[remove-messages-from-user.groovy](https://bitbucket.org/magnolia-cms/groovy-shell-scripts/src/master/examples/remove-messages-from-user.groovy)

```

| **1** | Messages removed for `superuser`. |

```
