# Image provider definition - 5 UI

Deprecated

> **Note:** Image provider definition has been deprecated since Magnolia 6.0. It is part of the Magnolia 5 UI framework.For the updated implementation, see Preview definition for Magnolia 6 UI instead.

Unresolved include directive in modules/ROOT/pages/Content-apps-5-UI/App-configuration-5-UI/Image-provider-definition-5-UI.adoc - include::6.2@product-docs:ROOT:partial$i/\_What-is-an-image-provider.adoc\[\]

![Image provider](6.2@product-docs:ROOT:thumbnail-portrait-images.png)

Here is an example image provider definition that uses default classes.

```yaml
imageProvider:
  class: info.magnolia.ui.imageprovider.definition.ConfiguredImageProviderDefinition
  imageProviderClass: info.magnolia.ui.imageprovider.DefaultImageProvider
  originalImageNodeName: jcr:content
```

| Property | Description |
| --- | --- |
| `class` | **required**<br><br>Image provider definition class that reads the configuration properties and can supply additional parameters to the image provider class. The definition class must implement the `ImageProviderDefinition` interface. Set the value to a fully-qualified class name. |
| `imageProviderClass` | **required**<br><br>Actual implementation that generates thumbnail images. The class must implement the `ImageProvider` interface. You can use `DefaultImageProvider`, which provides images for objects of MIME type `image/*` and delegates to the Imaging module to generate and store the images. |
| `imageExtension` | **optional**, *default is `png`*<br><br>Extension used to deliver the image. |
| `imagingServletPath` | **optional**, *default is `/.imaging`*<br><br>Path where the imaging servlet resides. |
| `originalImageNodeName` | **optional**, *default is `originalImage`*<br><br>Name of the JCR node where the original source image is stored. For example, if a file upload field on a form stores the image in a node named `photo` set the value to `photo`.<br><br>Here are the default image generator configurations that `DefaultImageProvider` uses. `portrait` is displayed at the bottom of the action bar and `thumbnail` in the thumbnail view.<br><br>![originalImage node](6.2@product-docs:ROOT:ADVT-img-provider-def.png) |
