## [](#_referring_to_a_content_type)Referring to a content type

Since the introduction of the [Content Types module](#6.2@product-docs:ROOT:Modules/List-of-modules/Content-Types-module.adoc), you can refer to a [content type definition](#6.2@product-docs:ROOT:Modules/List-of-modules/Content-Types-module/Content-type-definition.adoc) within an app descriptor.

This approach keeps the app descriptor very short. There is no need to specify the below mentioned [properties](#_properties). The content types and content apps frameworks automatically generate and register an app description based on the content type definition.

Example:

/content-type-examples/apps/tourGuides-app.yaml

[https://git.magnolia-cms.com/users/cmeier/repos/content-type-examples/raw/apps/tourGuides-app.yaml?at=master](https://git.magnolia-cms.com/users/cmeier/repos/content-type-examples/raw/apps/tourGuides-app.yaml?at=master)

You can tailor the app descriptor within the same file by adding additional configuration which will overwrite any configuration with the same name coming from the `!content-type` directive. Alternatively create a [decoration](#6.2@product-docs:ROOT:Developing/Definition-decoration.adoc).

tourGuides-app.yaml (with additional configuration)

[https://git.magnolia-cms.com/users/cmeier/repos/content-type-examples/raw/apps/tourGuides-app.yaml?at=master](https://git.magnolia-cms.com/users/cmeier/repos/content-type-examples/raw/apps/tourGuides-app.yaml?at=master)

For further details please see [Using Magnolia Content Types - Creating a content app based on the content type](#6.2@product-docs:ROOT:Developing/Content-Types-tutorial/Part-I-My-first-content-type.adoc#_creating_content_app_based_on_ct).

## [](#_properties)Properties

Here is an example descriptor for a `helloworld` app.

ui-helloworld-app/apps/helloworld.yaml

```js
appClass: info.magnolia.ui.app.helloworld.HelloWorldApp
class: info.magnolia.ui.api.app.registry.ConfiguredAppDescriptor
icon: icon-app
label: Hello World
subApps:
```

## [](#_permissions)Permissions

The permission to use an app is granted in the `permissions` configuration. The subnodes are roles. This allows you to provision the app to certain users in your organization.

In this example the helloworld app is provisioned to `travel-demo-editor` and `travel-demo-publisher` roles. The role property name (`editors`, `publishers`) is arbitrary - use any name you like. The value must be a valid role name.

ui-helloworld-app/apps/helloworld.yaml

```js
appClass: info.magnolia.ui.app.helloworld.HelloWorldApp
class: info.magnolia.ui.api.app.registry.ConfiguredAppDescriptor
icon: icon-app
label: Hello World
permissions:
  roles:
    editors: travel-demo-editor
    publishers: travel-demo-publisher
```
