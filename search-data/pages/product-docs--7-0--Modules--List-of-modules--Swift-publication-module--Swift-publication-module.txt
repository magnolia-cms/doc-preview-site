# Swift publication module

Unbundled: Extension

|     |     |
| --- | --- |
| **Edition** | DX Core |
| **License** | [MLA](https://www.magnolia-cms.com/mla.html) |
| **Issues** | [SWIFTPUB](https://magnolia-cms.atlassian.net/browse/SWIFTPUB) |
| **Maven site** | [Swift publication](https://nexus.magnolia-cms.com/#browse/browse:enterprise:info%2Fmagnolia%2Fswift%2Fpublication%2Fmagnolia-swift-publication%2F3.0.0-beta) |
| **Latest** | 3.0.0-beta |

Swift publication is a standalone enterprise module that can be used as a replacement for [DX Core publishing](../../Publishing-module/) in the Pages app. It stores versions in an external version store, such as S3. When an author publishes changes, the changes are sent to and stored in the external version store. Public instances then receive notifications to fetch the corresponding version from the version store.

REST communication is used between author and public instances, coordinating the information required for publishing. This allows instances to communicate asynchronously even if an instance is down or not reachable directly.

> **Important:** Swift publication currently manages publication for Pages only. Assets and other content types continue to use the standard DX Core publication mechanism by default. See Configuring the Assets app for more details.As a result, when configuring public instances you must define both sets of receiver properties:magnolia.publishing.receivers.* — DX Core publishing (used by Assets when Swift is installed, or by all content if Swift is not used).swift.publication.receivers.* — Swift publication (applies to Pages only by default).Define both for every public instance you configure so that Pages publish via Swift and other content types publish via DX Core publishing.

## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../../Developing/Bundles-and-webapps/):

Some steps below are required, and some parts are optional.

1.  Add the following mandatory dependencies.

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-api</artifactId>
      <version>3.0.0-beta</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |
    | **1** | Should you need to specify the module version, do it using `<version>`. |

    ```xml
    <dependency>
      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-versioning-ui</artifactId>
      <version>3.0.0-beta</version> (1)
    </dependency>
    ```

    |     |     |

2.  Add the S3 storage service dependency.

3.  Add REST communication between the author and the public instances dependency.

    ```xml

      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-rest</artifactId>
      <version>3.0.0-beta</version> (1)
    </dependency>
    ```

    |     |     |
    | --- | --- |

4.  If required, add a dependency for the Personalization module.

    ```xml

      <groupId>info.magnolia.swift.publication</groupId>
      <artifactId>magnolia-swift-publication-personalization</artifactId>
      <version>3.0.0-beta</version> (1)
    </dependency>
    ```

    |     |     |

    | **1** | It’s good practice to use the `version` variable from the parent POM (see in the last step), hence `{mgnlSwiftVersion}` would be here instead of `3.0.0-beta`. |

5.  Add the module version to the `properties` of the parent POM.

    Parent POM

    ```xml

    ```
