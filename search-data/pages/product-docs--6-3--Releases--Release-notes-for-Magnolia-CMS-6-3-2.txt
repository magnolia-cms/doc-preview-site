# Release notes for Magnolia CMS 6.3.2

[LTS release](../../../../support/) • Delivered on December 11, 2024 • Page updated on January 8, 2025 • Changelog: [14368](https://magnolia-cms.atlassian.net/issues/?filter=14368)

**Magnolia CMS 6.3.2** is a security and bug-fixing release primarily related to [known issues](../Release-notes-for-Magnolia-CMS-6.3.1/Magnolia-6.3.1-known-issues/) in the previous release.

> **Note:** See Upgrading to Magnolia 6.3 and Upgrading to Magnolia 6.3.1 and later before installing this version.

We keep the details of security fixes private in line with our [security policy](../../Administration/Security/Security-policy/). Contact our [Support team](../../../../support/) if you need more information.

## [](#_bug_fixes)Bug fixes

-   [BUILD-1299](https://magnolia-cms.atlassian.net/browse/BUILD-1299) Upgrade to commons-io `2.17.x` causes DAM app and Imaging module to break

-   [CFGUI-199](https://magnolia-cms.atlassian.net/browse/CFGUI-199) In-app search and filter don’t work correctly in Definitions app

-   [MAGNOLIA-9503](https://magnolia-cms.atlassian.net/browse/MAGNOLIA-9503) Classpath definitions can’t be overlayed with filesystem definitions

-   [MGNLRES-421](https://magnolia-cms.atlassian.net/browse/MGNLRES-421) In-app search and filter don’t work correctly in Resource Files app

-   [MGNLUI-9079](https://magnolia-cms.atlassian.net/browse/MGNLUI-9079) Search field is missing in some apps after converting the definitions

-   [MGNLUI-9095](https://magnolia-cms.atlassian.net/browse/MGNLUI-9095) Switching variants replaces the target variant with the current value without clicking Save Changes

-   [TASKS-64](https://magnolia-cms.atlassian.net/browse/TASKS-64) In-app search and filter don’t work correctly in Tasks app
