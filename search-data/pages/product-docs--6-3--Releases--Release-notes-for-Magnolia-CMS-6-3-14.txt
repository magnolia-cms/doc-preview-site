# Release notes for Magnolia CMS 6.3.14

[LTS release](../../../../support/) • Delivered on August 14, 2025 • Page updated on December 1, 2025 • Changelog: [19318](https://magnolia-cms.atlassian.net/issues/?filter=19318)

**Magnolia 6.3.14** is primarily a bug-fixing and security release that also externalizes the `ipSecurity` filter to the MicroProfile config to ensure no risk of lockout, as it does not rely on JCR accessibility. Additionally, the Assets app now allows users to select folders for bulk downloads and displays a progress bar during the upload of multiple assets.

We keep the details of security fixes private in line with our [security policy](../../Administration/Security/Security-policy/). Contact our [Support team](../../../../support/) if you need more information.

#### [](#_security_hardening_for_delivery_endpoints)Security hardening for delivery endpoints

[MGNLREST-875](https://magnolia-cms.atlassian.net/browse/MGNLREST-875) introduces automatic encoding of content output for REST delivery endpoints. Content escaping is now enabled by default on delivery endpoints so that the resulting JSON can be parsed correctly by recipients.

> **Caution:** This is a breaking change (if using the default configuration).With escaping enabled by default now, this change affects content containing specific characters such as <, >, &, ", ', or /. The default escaping may break rendering in Single Page Applications (SPAs) using the Delivery API.To avoid issues:You can disable this escaping via configuration to retain the previous behavior.Alternatively, ensure your client-side application handles the escaped HTML content.

See also the new `magnolia.delivery.escaping` property on the [Configuration management](../../Administration/Architecture/Configuration-management/) page for more details.
