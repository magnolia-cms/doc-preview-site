## [](#_installing_with_maven)Installing with Maven

Maven is the easiest way to install the module. Add the following to your [bundle](../../../../../RNs-6-4-3/Developing/Bundles-and-webapps/):

```xml
<dependency>
  <groupId>info.magnolia.advancedcache</groupId>
  <artifactId>magnolia-advanced-cache-dpc</artifactId>
  <version>{magnolia-advanced-cache-64}</version> (1)
</dependency>
```

|     |     |
| --- | --- |
| **1** | Should you need to specify the module version, do it using `<version>`. |

## [](#_uninstalling)Uninstalling

Shut down Magnolia, remove `magnolia-advanced-cache-dpc.jar` from `WEB-INF/lib` and start up Magnolia again.

| Field | Description | Available values | Default value |
| --- | --- | --- | --- |
| `before` | A text to be injected before dynamic components. Occurrences of *\({targetPath}* will be replaced with the JCR path to this component. Occurrences of *\){TTL}* will be replaced with *DynamicFragmentDefinition#getTtl* . | any string | - |
| `after` | A text to be injected after dynamic components. Occurrences of *\({targetPath}* will be replaced with the JCR path to this component. Occurrences of *\){TTL}* will be replaced with *DynamicFragmentDefinition#getTtl* . | any string | - |
| `targetPath` | The URI format of dynamic components. | RELATIVE_TO_SERVER_ROOT RELATIVE_TO_CONTEXT_ROOT RELATIVE_TO_CURRENT_PAGE | RELATIVE_TO_CONTEXT_ROOT |
| `skipRendering` | Skip rendering of dynamic components if **true** , otherwise renders current content of the component.
