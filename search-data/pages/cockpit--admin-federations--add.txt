# Add a Federation

1.  In the Cockpit, go to **Admin > Federations**.

2.  Enter a [Display name](#displayName) for your Federation. This autofills the [Alias](#alias) field.

3.  Check the box for **Enabled** if you want to enable the Federation.

4.  Select the [type](#type) of the Federation. This is either OIDC or SAML.

    -   OIDC

    -   SAML


    The following are required:

    1.  Enter the [Authorization URL](#_authorization_url).

    2.  Enter the [Token URL](#tokenUrl).

    3.  Enter the [Client ID](#clientId).

    4.  Enter the [Client secret](#clientSecret).


    The remainig are optional but recommended:

    1.  Enter the [Logout URL](#logoutUrl).

    2.  Enter the [User info URL](#userInfoUrl).

    3.  Enter the [Issuer](#issuer).

    4.  Check the box for [Validate signature](#validateSignature) if you want to validate the signature of the IDP token.

    5.  Check the box for [Use JWKs URL](#useJwksUrl) if you want to use the JWKs URL to validate the signature of the IDP token.

    6.  Enter the [JWKs URL](#jwksUrl).


    The following are required:

    1.  Enter the [SAML Metadata](#samlMetadata).


    The remaining are optional but recommended:

    1.  Enter the [Name ID Policy Format](#nameIdPolicyFormat).

    2.  Enter the [Principal Type](#principalType). Use `NameID`.


5.  Under **Advanced settings**, choose the [Sync Mode](#syncMode). This is either **Force on every login** or **Import only on first login**.

6.  Check the box for [Trust Email](#trustEmail) if you want to trust the email of the user.
