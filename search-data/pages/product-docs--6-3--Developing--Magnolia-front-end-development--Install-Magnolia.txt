# Install Magnolia

This page provides instructions on how to do a basic installation of Magnolia using the Magnolia CLI *v4*. You should do this before starting the [Hello Magnolia for front-end developers](../Hello-Magnolia-for-front-end-developers/) tutorial.

## [](#_get_java)Get Java

Magnolia needs at least a **Java Runtime Environment (JRE)** to run. Check if thereâ€™s a version of Java already installed on your computer by opening the terminal or command prompt and typing `java -version`. If the system reports a version number, Java is installed on your computer.

> **Important:** See Certified stack to confirm that the version installed is supported.For local development, use 17.0.12-zulu. Example installation using SDKMAN:sdk install java 17.0.12-zulu

-   Windows

-   Mac

-   Linux


On Windows, you need a Java SE *Development* Kit (JDK). The Java *Runtime* Environment (JRE) is not enough because the Tomcat application server does not recognize it.

-   JRE is for users who run Java programs on their computers.

-   JDK is for developers who write Java-based applications.


[Download](http://www.oracle.com/technetwork/java/javase/downloads/index.html) and install JDK. By default, JDK is installed at `C:\Program Files\Java\jdk-<version>\`.

For Mac, you need to download or update to Java `17` or higher. After reviewing and agreeing to the terms of the license agreement, download the file, then double-click it to launch the installation wizard and follow the installation instructions.

The installation directory varies from one Linux system to another. On Debian-based distributions, JREs or JDKs are usually installed in `/usr/lib/jvm/`.

## [](#_get_node_js)Get node.js

Magnolia CLI is an [npm](https://www.npmjs.com/) package providing a command line interface (CLI) tool to set up and facilitate light development with Magnolia. The Magnolia CLI tool runs on [Node.js](https://nodejs.org/). If you do not have Node.js installed, go to [Node.js](https://nodejs.org/) and download and install the latest LTS version.

To check the version of your node installation, run the following command in a shell:

node -v

## [](#_start_magnolia_with_cli)Start Magnolia with CLI

Go to the parent directory of the `apache-tomcat` directory and execute the [Magnolia CLI](#magnolia-cli:ROOT:index.adoc) `start` command:

mgnl start

When starting for the first time, Magnolia runs a web update and automatically installs all its modules.

## [](#_login)Login

Go to [`http://localhost:8080/magnoliaAuthor`](http://localhost:8080/magnoliaAuthor) and sign in as a `superuser` :

-   Username: `superuser`

-   Password: `superuser`


Magnolia is ready to use and gives you a list of suggestions to get started.

> **Note:** superuser is a system administrator account that has permissions to do everything. It is useful for testing.
