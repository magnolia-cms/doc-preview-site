# Magnolia CMS Documentation

## About Magnolia DXP

Magnolia is a composable digital experience platform (DXP). It helps organizations manage content, deliver experiences, and integrate with other systems.

**Offerings:**
- **DX Core** – Self-hosted or on-premises Magnolia (product-docs, versions 6.2, 6.3, 6.4, 7.0).
- **DX Cloud** – Managed Magnolia on PaaS; documentation under paas/.
- **Headless**, **CLI**, **Modules** – Additional product areas covered in this site.

## Audience

This documentation serves **developers** (templating, APIs, headless, modules), **authors** (content creation, assets, workflows), **administrators** (security, instances, configuration), and **marketers** (personalization, campaigns, analytics).

## Key user paths

1. **Getting started** – Installation, Hello Magnolia, first page.
2. **Authoring** – Creating and editing content, assets, versioning, search.
3. **Developing** – Templating, APIs, headless/SPA, best practices.
4. **Administration** – Security, IAM, instances, monitoring, backup.
5. **Modules & apps** – List of modules/apps, configuration, extensions.
6. **DX Cloud (PaaS)** – Deploying and operating Magnolia on PaaS, Cockpit.

## Output guidelines for assistants

- Prefer **step-by-step** instructions when explaining procedures.
- **Provide links** to the relevant documentation pages (use the URLs from this manifest or the linked .txt content).
- **Do not invent** endpoints, configuration keys, or content that is not present in the documentation.
- When **uncertain**, say so and point to the most relevant doc section.

## Citation style

Cite sources as: [Page or section title](url). Prefer linking to the canonical docs page (e.g. the HTML URL for the topic) when giving the user a link; the .txt files below are for loading full text.
